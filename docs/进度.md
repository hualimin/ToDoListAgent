> **子系统 A 待办的 2 个已知项（最终审查标记，非阻塞，建议 A 开工即修）**：
> 1. `app/src/lib/taskViews.ts` 的 `withinRange`：`week`/`month` 当前是"过去窗口"（逾期），非"未来窗口"——与"本周/本月"直觉相反，默认"今日"视图对新任务（无 due_at）为空。需明确范围语义（未来窗口？无截止任务如何归桶？）。
> 2. `InMemoryTaskRepository` 的 `board_order = this.items.size`：软删后 size 不减，新建会复用已存在的 order → 排序并列。子系统 A 的拖拽看板需要稳定/单调/可重排的 order（改用 max+1 或 float）。
> 另：原生 SqliteTaskRepository 为 stub（真机阶段补真实 SQL）；反思/学习页为模拟数据（B/C 接真实数据）。
